Portal — Lo-Fi Wireframes

v1 design reference for the contextus client portal. Used as the canonical visual source by Claude Code agents implementing the frontend and backend.

VERSION 1.0 · APRIL 2026 · INTERNAL

What this is. Ten lo-fi wireframes covering every screen and state in the contextus portal v1. Grayscale, no real copy, no icons — the point is layout and information density, not pixel polish.

How to use it. Each wireframe is annotated with the endpoints that feed it (see BACKEND-SPEC-PORTAL-V1.md) and the route it lives at (see FRONTEND-SPEC-PORTAL-V1.md). Implementation agents reference these wireframes during PR work. Do not redesign — port these layouts using the brand tokens from contextus-widget-design-guideline.html.

Out of scope for v1. Analytics tab is sidebar-disabled. Self-serve site creation, recrawl, and Q&A edit/delete are deferred to v2.

Wireframe 10
Add Q&A modal

Triggered from the Knowledge tab — either via "+ add Q&A" or "+ answer this" on a gap card (which pre-fills the question field). Two fields, one warning about regeneration. Matches the additive-only constraint of the enrich endpoint.
